Javascript and the DOM

HTML and CSS

While HTML and CSS are two very different programming languages, both need each other to work. HTML is your content on the page. Your words, your links, your images etc. Think of it like the rooms in a house. Its the walls, the ceiling and windows that you see. When you first build a house (or website) these things look plain, barebone so to speak. This is where CSS comes in. CSS tells the HTML what to look like. It sets sizes of images, fonts, colours and much, much more. In regards to our analogy of the house, CSS is like your interior design. Its the paint on your walls, the curtains you hang or ugly lamp your aunt gave you last christmas.

Control Flow and Loops

Control flow is the order or sequence in which parts of your code or programme activate. A loop is exactly as the name suggests. Where a programme completes a task over and over untill is reaches the condition that it has been told to stop at, eg: a set amount of times.

A great example of control flow in real life is making a coffee. There is a process to this. You first need to decide if you want a coffee. Then you need to go to the kitchen. You then go through the process of gringing your beans, tamping your shout, loading it in the machine. You extract your shot and then decide if you want milk (Yes? No?). Are you having sugar? Once you are done you then drink the coffee. The loop then begins again, as your crippling caffiene addiction kicks in and your drag yourself to the kitchen for another hit. And again and again until you have had the number you permit yourself a day or until you have a severe case of the coffee jitters and can't type (wait, is this just me?).

The DOM

The Document Object Model (DOM) shows how different parts of your code are related to each other. Similar to a family tree, elements of code follow a structure of parent-child-sibling. Each branch of the tree has a node at the end, which conatins your data or content.

Everytime you load a new webpage an DOM is created. The DOM is what defines how the document is changed and accessed. Your browser uses it to figure out what it needs to render on your screen(the veiwport) and javascript programmes use it to help manipulate the content of a page (eg. make it do stuff).

Arrays and Objects

A variable in Javascript is basically a container for data. An array is a variable that can contain more than one value at a time. They can contain many values or just a couple and span just one line, or many.

To access an array you use bracket notation. Its worth noting that arrays are numbered in order from zero onwards. Meaning that the first item in your array is 0 and not 1.

An Object is a type of variable that can contain other variables that have their own value (or piece of data/information). The values are written as name:value pairs (name and value separated by a colon).

To access data in an object you can use two different methods, either dot or bracket notation, using the name of the variable and the name of the value.

Functions

A functions is a piece of code that is written to perform a particular task, especially one that is required to be repeated. This is useful because they are able to be reused over and over.
